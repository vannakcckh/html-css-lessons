01 — Overview

Transitions vs. Animations

Both let you add motion to elements, but they serve different purposes:

Featuretransitionanimation
TriggerState change (hover, focus, class)Runs automatically or on class add
KeyframesOnly start → endFull @keyframes control
LoopNoYes (infinite or n times)
DirectionReverses on un-triggerConfigurable (alternate, reverse…)
ComplexitySimple 2-state changesMulti-step complex sequences
02 — Transitions

The transition Property

Transitions smoothly animate a CSS property from one value to another when triggered by a state change.

/* Longhand */
.box {
  transition-property:  background;
  transition-duration:  0.3s;
  transition-timing-function: ease;
  transition-delay:    0s;
}

/* Shorthand (property | duration | easing | delay) */
.box {
  transition: background 0.3s ease 0s;
}

/* Multiple properties */
.box {
  transition:
    background 0.3s ease,
    transform 0.5s cubic-bezier(0.34, 1.56, 0.64, 1),
    opacity 0.2s linear;
}

/* Transition ALL properties (use with caution) */
.box { transition: all 0.3s ease; }

04 — @keyframes

CSS Animations with @keyframes

Animations let you define multiple steps using @keyframes, then apply them with the animation property.

/* 1. Define the keyframes */
@keyframes slide-in {
  from { transform: translateX(-100px); opacity: 0; }
  to   { transform: translateX(0);      opacity: 1; }
}

@keyframes color-cycle {
  0%   { background: red; }
  33%  { background: blue; }
  66%  { background: green; }
  100% { background: red; }
}

/* 2. Apply the animation (shorthand) */
/*    name | duration | easing | delay | iterations | direction | fill-mode | play-state */
.element {
  animation: slide-in 0.5s ease-out 0s 1 normal forwards running;
}

/* animation-direction */
.ping   { animation-direction: normal; }         /* → always forward */
.rewind { animation-direction: reverse; }        /* ← always backward */
.yo-yo  { animation-direction: alternate; }       /* →← ping-pong */

/* animation-fill-mode — what happens AFTER the animation */
.appears {
  animation: fade-in 0.5s ease forwards; /* stays visible after */
}

/* Pause on hover */
.spinner      { animation: spin 1s linear infinite; }
.spinner:hover{ animation-play-state: paused; }

06 — Transforms

The transform Property

Transforms change the visual presentation of an element without affecting layout. They pair perfectly with transitions and animations.

/* 2D Transforms */
transform: translateX(50px);        /* move horizontally */
transform: translateY(-20px);       /* move vertically */
transform: translate(50px, -20px);  /* move both axes */
transform: rotate(45deg);           /* rotate clockwise */
transform: scale(1.5);              /* scale up 150% */
transform: scale(0.5, 2);           /* scale x, y independently */
transform: skewX(20deg);            /* horizontal skew */

/* Combine multiple transforms (space-separated) */
transform: rotate(30deg) scale(1.2) translateX(10px);

/* 3D transforms */
transform: rotateY(180deg);
transform: perspective(600px) rotateX(30deg);
transform-origin: center;          /* pivot point: top left, 50% 50%, etc. */

07 — Practical Examples

Real-world Patterns

/* Loaders */
@keyframes spin { to { transform: rotate(360deg); } }
.spinner {
  border: 3px solid #eee;
  border-top-color: steelblue;
  border-radius: 50%;
  animation: spin 0.8s linear infinite;
}

/* Skeleton / shimmer */
@keyframes shimmer {
  from { background-position: -200% center; }
  to   { background-position: 200% center; }
}
.skeleton {
  background: linear-gradient(90deg, #eee 25%, #ddd 50%, #eee 75%);
  background-size: 200% 100%;
  animation: shimmer 1.5s linear infinite;
}

/* Card flip */
.scene       { perspective: 600px; }
.card        { transform-style: preserve-3d; transition: transform 0.7s; }
.card:hover  { transform: rotateY(180deg); }
.front, .back { backface-visibility: hidden; }
.back        { transform: rotateY(180deg); }

08 — Performance

Performance Best Practices

/* ✅ GPU-accelerated — smooth at 60fps */
transform: translate(), scale(), rotate();
opacity: 0..1;

/* ❌ Avoid — triggers layout reflow (expensive) */
width, height, top, left, margin, padding;

/* Hint the browser to prepare a layer in advance */
.will-animate {
  will-change: transform, opacity;  /* use sparingly */
}

/* Respect reduced motion preference (accessibility!) */
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
09 — Quick Reference

Cheatsheet

Transition Shorthand

transition: property duration easing delay;

transition: background 0.3s ease 0s;

transition: all 0.3s ease;

Animation Shorthand

animation: name duration easing delay count dir fill;

animation: spin 1s linear infinite;

animation: fade-in 0.5s ease forwards;

Common Keyframes

from { } to { } — simple 2-step

0% { } 50% { } 100% { } — percentages

Multiple properties per step ✓

GPU-Safe Properties

transform: translate/scale/rotate

opacity: 0 → 1

filter: blur/brightness

Timing Functions

linear — constant speed

ease-out — fast → slow (most natural)

cubic-bezier(x1,y1,x2,y2) — custom

Fill Mode

forwards — keeps end state

backwards — applies start during delay

both — forwards + backwards
